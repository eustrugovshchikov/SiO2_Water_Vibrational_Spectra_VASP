# SiO2_Water_Vibrational_Spectra_VASP

**SiO2_Water_Vibrational_Spectra_VASP** is a Python-based tool for computing vibrational spectra from molecular dynamics trajectories generated with VASP. It is primarily designed for simulations using machine-learned force fields (MLFF) in VASP, but is equally applicable to ab initio molecular dynamics.

The code enables analysis of OH and CH₃ functional groups as well as water molecules in silica-based systems, including both bulk water and near-surface environments. It focuses on extracting localized vibrational modes, such as O–H stretching in surface hydroxyls and interfacial or bulk water.

Vibrational spectra are obtained from the Fourier transform of time-dependent bond-length fluctuations (or their autocorrelation functions), providing a direct and efficient route from MD trajectories to frequency-domain information.

Recent updates significantly reduce computational cost and memory usage by avoiding full trajectory parsing (e.g., large XDATCAR files) and instead processing only the required subsets of data. This leads to substantially faster execution and improved scalability for long simulations, which is particularly important for MLFF-based trajectories.

The tool is particularly suited for studies of silica–water interfaces, surface functionalization, and related systems where localized vibrational signatures are of interest.

## Theory

This tool extracts vibrational frequencies from molecular dynamics (MD) trajectories generated by VASP by analyzing time-dependent fluctuations of selected bond lengths. The approach is based on transforming these time-domain signals into the frequency domain using the Fourier transform.

1. **MD Trajectory Output**  
   - Perform a VASP MD simulation (MLFF or ab initio) and configure XDATCAR to record ionic positions at each timestep. 
   - Trajectories should include all relevant atoms (Si, O, H, C) for silica, functional groups, and water.

2. **Signal Preparation**  
   - Select atoms corresponding to OH groups, CH₃ groups, and/or water molecules using their indices.  
   - Construct time series of bond-length fluctuations (e.g., O–H, C–H) for each selected pair.
   - Optionally, restrict analysis to subsets of frames to reduce computational cost for large trajectories.

3. **Time-to-Frequency Transformation**  
   - Compute the Fourier transform of the bond-length time series (or their autocorrelation functions).  
   - This yields frequency-domain representations where peaks correspond to vibrational modes.

4. **Spectrum Construction**  
   - Convert Fourier amplitudes into power spectra.
   - Average over equivalent bonds and normalize intensities.
   - Output the resulting vibrational spectrum as text data for further analysis or plotting.

## Global Variables

Control analysis behavior via user-defined variables in your environment or `submit.sh` file.

```bash
# ***** USER-DEFINED VARIABLES *****
# Set simulation frame range (modify these numbers before submission)
START_FRAME=5000
FINISH_FRAME=100000
# ***********************************

# Based on the command-line argument (1,2,3,4,5), the script outputs:
#  1) Mode 1 (only OH): Bonds: H–O and O–Si, Angle: H–O–Si.
#  2) Mode 2 (only CH3): Bonds: C–H (for each H in CH3) and C–Si;
#       Angles: all H–C–H combinations and H–C–Si angles.
#  3) Mode 3 (only surface angles): Angles at the Si center between any two functional groups.
#  4) Mode 4 (all surface groups): Both groups’ bonds/angles plus the surface angles.
#  5) Mode 5 (only water): Bonds: H–O (only once per bond) and angle: H–O–H.

# Example invocation for Mode 4:
echo "Running find_indices.py to extract H atom indices..."
python find_indices.py 4


Input and output filenames (optional overrides)
INPUT_XDATCAR = "XDATCAR"
OUTPUT_SPECTRUM = "output_average.dat"

```

## Examples

To run an example:

1. **Download and unpack**  
   - Download the `EXAMPLES` folder from this repository.  
   - Unpack the `XDATCAR` file into the same directory.

2. **Edit the SLURM script**  
   - Open the provided `submit.sh` script.  
   - Adjust the `#SBATCH` directives to match your cluster’s account, partition, node, and time limits.  
   - Ensure you load the correct Python module for your environment.

```bash
#!/bin/bash -x 
#SBATCH --account=dcc70
#SBATCH --partition=mylemta
#SBATCH --job-name=FFT
#SBATCH --output=slurm-%x.%N.%j.out 
#SBATCH --error=slurm-%x.%N.%j.err 
#SBATCH --nodes=1
#SBATCH --ntasks=32
#SBATCH --ntasks-per-node=32
#SBATCH --cpus-per-task=1
#SBATCH --time=3-01:00:00
#SBATCH --exclude=cnd[01-04,06-12]

# Load the Python module.
module load python/3.6/anaconda
```
3. **Submit the job**
```bash
sbatch submit.sh
```

After the script runs, the results will be written to `output_average.dat`.
